## 一、当前配置

```json5
{
  session: {
    dmScope: "per-peer"  // 默认配置，无显式 reset 段
  }
}
```

由于未配置 `reset` 段，系统使用内置默认值：
- **默认模式**：`daily`（每天重置）
- **默认时间**：凌晨 4:00（BJT / 本地时区）
- **默认触发命令**：`/new`、`/reset`

## 二、配置参数详解

### 2.1 主重置策略 `session.reset`

```json5
{
  session: {
    reset: {
      mode: "daily",      // daily | idle
      atHour: 4,          // 仅 daily 模式：凌晨 4 点重置
      idleMinutes: 60,    // 仅 idle 模式：60 分钟无互动后重置
    }
  }
}
```

| 字段 | 可选值 | 说明 |
|------|--------|------|
| `mode` | `"daily"` | 每天固定时间重置 |
| `mode` | `"idle"` | 会话空闲超过指定分钟数后重置 |
| `atHour` | `0-23` | daily 模式下的重置小时（本地时区） |
| `idleMinutes` | 正整数 | idle 模式下的空闲超时分钟数 |

**优先级**：当 `daily` 和 `idle` 同时配置时，**先到期者触发**。

**重置判定机制**：
- `daily` 模式：基于会话的 `sessionStartedAt`（会话开始时间）
- `idle` 模式：基于会话的 `lastInteractionAt`（最后互动时间）
- ⚠️ 后台写操作（心跳、cron 唤醒、exec 通知、网关记账）不会刷新 idle 计时器

### 2.2 按会话类型覆盖 `session.resetByType`

```json5
{
  session: {
    resetByType: {
      thread: { mode: "daily", atHour: 4 },
      direct: { mode: "idle", idleMinutes: 240 },
      group:  { mode: "idle", idleMinutes: 120 },
    }
  }
}
```

| 类型 | 说明 | 别名 |
|------|------|------|
| `direct` | 私聊会话 | 兼容旧版 `dm` |
| `group` | 群组会话 | — |
| `thread` | 线程会话 | — |

可针对不同聊天类型设置不同的重置策略。

### 2.3 会话作用域 `session.dmScope`

```json5
{
  session: {
    dmScope: "main"  // main | per-peer | per-channel-peer | per-account-channel-peer
  }
}
```

| 值 | 说明 |
|----|------|
| `main` | 所有私聊共用同一会话 |
| `per-peer` | 按发送者隔离会话（跨频道） |
| `per-channel-peer` | 按频道+发送者隔离（推荐多用户） |
| `per-account-channel-peer` | 按账号+频道+发送者隔离（推荐多账号） |

### 2.4 手动重置触发器 `session.resetTriggers`

```json5
{
  session: {
    resetTriggers: ["/new", "/reset"]
  }
}
```

内置聊天命令：
- `/new` — 新建会话（清除上下文）
- `/reset` — 重置当前会话

## 三、存档保留策略 `session.maintenance`

```json5
{
  session: {
    maintenance: {
      mode: "warn",              // warn | enforce
      pruneAfter: "30d",         // 会话条目保留期限
      maxEntries: 500,           // sessions.json 最大条目数
      resetArchiveRetention: "30d",  // .reset.* 归档文件保留时间
      maxDiskBytes: "500mb",     // 会话目录磁盘硬上限
      highWaterBytes: "400mb",   // 清理目标水位线
    }
  }
}
```

- `resetArchiveRetention`：控制 `.reset.` 后缀的归档文件保留时间（默认等于 `pruneAfter`，设为 `false` 可关闭归档）
- 每次重置时，旧会话会保存为 `{sessionId}.jsonl.reset.{timestamp}` 然后新建会话

## 四、重置事件生命周期

```
重置触发（daily / idle / 手动）
    ↓
┌──────────────────────────────────────┐
│ 1. 检查是否满足重置条件               │
│ 2. 当前会话 → 归档为 .reset. 文件     │
│ 3. 创建新会话                         │
│ 4. 新会话首次运行时加载启动 prelude   │
│    （如果是模型参与的重置）            │
└──────────────────────────────────────┘
```

- `/new` 和 `/reset` 命令**不触发模型调用**（直接清空上下文）
- 自动 daily reset 会触发启动 prelude（加载最近 memory 等上下文）

## 五、相关 CLI 命令

```bash
# 手动重置
openclaw reset --scope config+creds+sessions --yes --non-interactive

# 查看会话状态
openclaw status

# 查看会话文件
ls ~/.openclaw/agents/main/sessions/
```
