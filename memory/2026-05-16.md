### 修复详情

#### 漏洞 18：ReDoS in auto-capture.ts
**位置**：`src/hooks/auto-capture.ts:144`
**问题**：
```typescript
const excludePatterns = (getProp(config, "capture.excludePatterns", []) as string[])
  .map(p => { try { return new RegExp(p, "i"); } catch { return null; } })
  .filter((r): r is RegExp => r !== null);
```
**攻击场景**：攻击者在配置中设置 `excludePatterns: ["(a+)+$"]`，当 `fullText` 包含大量 'a' 时，`pattern.test(fullText)` 会导致灾难性回溯，CPU 长时间占满。

**修复**：增加正则长度限制过滤：
```typescript
const MAX_PATTERN_LENGTH = 200;
const excludePatterns = (getProp(config, "capture.excludePatterns", []) as string[])
  .filter(p => typeof p === "string" && p.length <= MAX_PATTERN_LENGTH)
  .map(p => { try { return new RegExp(p, "i"); } catch { return null; } })
  .filter((r): r is RegExp => r !== null);
```

#### 漏洞 19：SSRF 重定向绕过 in llm-client.ts / embedding.ts
**位置**：`src/utils/llm-client.ts:194` / `src/utils/embedding.ts:105`
**问题**：经复核，`fetch` 调用已设置 `redirect: "manual"`，重定向绕过已在 batch-9/batch-10 修复。此条目为复核确认，非本轮新发现。
