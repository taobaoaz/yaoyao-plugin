## 2026-05-20 17:44-18:00 yaoyao ↔ xiaoyiclaw 共存模式适配

### 背景
用户更新了 xiaoyiclaw（https://cnb.cool/llm-memory-integrat/xiaoyiclaw），要求 yaoyao-plugin 与它并行运行而非替代它。

### v1 实现（commit 458e894）

**新增文件：**
- `utils/coexistence.ts` — UDS socket 检测，模式状态（standalone/coexist）
- `utils/claw-bridge.ts` — 短连接 UDS JSON-RPC 桥，每次调用新建 socket

**修改文件：**
- `entry/index.ts` — 启动时检测 claw-core 存在，日志标识当前模式
- `hooks/auto-capture.ts` — `skipHeavyLayers` 标志，共存模式跳过 L1/L2 写入
- `hooks/auto-recall.ts` — `clawBridge.recall()` 优先，本地结果补充

**v1 缺陷：**
- 每次召回新建 socket，开销大
- 仅启动时检测，无法感知运行时 claw-core 启停
- 串行查询（本地 → claw-core）延迟高

### v2 优化（commit bd532cf）

| 模块 | v1 问题 | v2 改进 |
|------|--------|---------|
| **claw-bridge.ts** | 每次召回新建 socket | 持久连接 + 自动重连 + 请求队列(max 3 in-flight) + 30s 健康检查 |
| **coexistence.ts** | 仅启动时检测 | 30s 自动刷新 + 状态变化事件回调 + `YAOYAO_COEXIST_MODE` 环境变量强制覆盖 |
| **auto-capture.ts** | 共存模式完全静默 | 可配置转发到 claw-core，失败自动回退本地 |
| **auto-recall.ts** | 串行查询 | **并行查询**（local + claw-core 同时）+ 智能合并去重 + 超时降级 |

**关键新能力：**
- `YAOYAO_COEXIST_MODE=disabled` — 即使检测到 claw-core 也强制独立运行
- 运行时 claw-core 突然消失 → 自动切回 standalone，不丢消息
- 运行时 claw-core 突然出现 → 自动切到 coexist，平滑过渡
