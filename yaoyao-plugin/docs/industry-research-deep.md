# AI 记忆系统深度研究报告

## 一、Mem0 深度分析

### 核心架构
Mem0 采用 **extract-update-retrieve** 管道：

```
对话 → 提取器(Extract) → 记忆池 → 整合器(Consolidate) → 图/向量存储 → 检索器(Retrieve)
```

**关键创新**:
1. **动态提取**: 不是保存整段对话，而是提取原子事实（atomic facts）
2. **图增强**: 基础版用向量存储，增强版用图结构捕获实体关系
3. **整合机制**: 新记忆触发旧记忆更新，避免重复

### 与 yaoyao 对比

| 维度 | Mem0 | yaoyao |
|------|------|--------|
| 存储粒度 | 原子事实 | L0 原始对话 + L1 结构化 |
| 图结构 | ✅ 显式图（实体-关系） | ❌ 无 |
| 整合策略 | 动态更新旧记忆 | 时间衰减 + 访问频率 |
| 去重 | 语义去重 + 图合并 | Jaccard + 哈希 |
| 延迟 | p95 降低 91% | 未测试 |
| 基准 | LOCOMO | 无 |

### yaoyao 可借鉴
- **原子提取**: 将 L1 从"结构化索引"升级为"原子事实池"
- **图关系**: 在 scene_blocks 间建立 `updates` / `related` 边
- **动态整合**: 新记忆触发旧记忆上下文更新

## 二、A-Mem 深度分析

### 核心架构
A-Mem 采用 **Zettelkasten 卡片盒方法**：

```
新记忆 → 生成完整笔记（上下文+关键词+标签） → 分析历史记忆 → 建立链接 → 触发旧记忆更新
```

**关键创新**:
1. **动态索引**: 不是预定义分类，而是基于语义相似性动态链接
2. **记忆演化**: 新记忆整合时，触发旧记忆的上下文重述
3. **Agentic 决策**: LLM 决定如何组织、链接、更新

### Zettelkasten 原则
- **原子性**: 每条笔记一个想法
- **链接性**: 笔记间双向链接
- **演化性**: 笔记随时间精炼

### 与 yaoyao 对比

| 维度 | A-Mem | yaoyao |
|------|-------|--------|
| 组织方式 | Zettelkasten 动态链接 | 时间线 + 场景分组 |
| 记忆演化 | ✅ 上下文重述 | ❌ 静态保留 |
| 链接类型 | 语义相似 + 时间 + 因果 | 仅时间关联 |
| 决策方式 | Agentic（LLM 决定） | 规则驱动 |

### yaoyao 可借鉴
- **双向链接**: 记忆间建立 `related_to` / `supersedes` 关系
- **上下文重述**: 定期用 LLM 精炼旧记忆的表述
- **动态分组**: 基于语义而非固定时间窗口

## 三、Supermemory 深度分析

### 核心架构
Supermemory 采用 **三层混合搜索**:

```
用户查询 → BM25(FTS5) → 向量相似度 → 图遍历 → MMR 重排序 → 注入上下文
```

**关键创新**:
1. **图遍历增强**: 种子节点 → 多跳邻居 → 关系证据
2. **MMR 重排序**: 平衡相关性和多样性（λ=0.7）
3. **用户画像**: 长期画像 + 近期记忆双轨注入
4. **生命周期管理**: 过期清理 + 别名合并 + 向量回填

### 混合搜索权重
| 组件 | 权重 | 作用 |
|------|------|------|
| 向量相似度 | 0.5 | 语义匹配 |
| BM25 关键词 | 0.3 | 精确匹配 |
| 图遍历 | 0.2 | 关联发现 |

### 与 yaoyao 对比

| 维度 | Supermemory | yaoyao |
|------|-------------|--------|
| 搜索融合 | 三轨加权融合 | FTS5 + 向量 RRF |
| 图遍历 | ✅ 2 跳深度 | ❌ 无 |
| MMR 多样性 | ✅ | ❌ 仅 Top-K |
| 用户画像 | ✅ 长期+近期 | ✅ persona.md |
| 别名处理 | ✅ 实体别名归一 | ❌ 无 |
| 过期策略 | ✅ 90 天衰减 | ✅ 时间衰减 |

### yaoyao 可借鉴
- **图遍历**: 在搜索结果上做 1-2 跳关联扩展
- **MMR**: 引入多样性，避免重复记忆
- **别名归一**: "张三"/"小张" → 同一实体
- **画像双轨**: 区分长期偏好和近期上下文

## 四、MAGMA 深度分析

### 核心架构
MAGMA 采用 **多图正交分解**:

```
记忆 → 语义图 → 时间图 → 因果图 → 实体图
查询 → 策略路由 → 选择性遍历 → 结构化上下文
```

**关键创新**:
1. **正交分解**: 同一记忆在 4 个独立图中表示
2. **策略路由**: 根据查询类型选择遍历哪个图
3. **透明推理**: 检索路径可解释

### 四图定义
| 图类型 | 边含义 | 查询适配 |
|--------|--------|----------|
| 语义图 | 主题相似 | 概念查询 |
| 时间图 | 先后/同时 | 时序查询 |
| 因果图 | 导致/依赖 | 原因查询 |
| 实体图 | 共现/关联 | 实体查询 |

### 与 yaoyao 对比

| 维度 | MAGMA | yaoyao |
|------|-------|--------|
| 记忆视图 | 4 个正交图 | 单一分层 |
| 查询适配 | ✅ 策略路由 | ❌ 统一搜索 |
| 可解释性 | ✅ 遍历路径 | ❌ 黑盒分数 |
| 复杂度 | 高（4 图维护） | 低 |

### yaoyao 可借鉴
- **查询分类**: 判断查询类型（概念/时序/因果/实体）
- **选择性搜索**: 时序查询优先时间衰减权重
- **路径解释**: 返回记忆时附带"为什么检索到"

## 五、Memory Unified 深度分析

### 核心架构
Memory Unified 采用 **双后端 + 技能学习**:

```
USMD SQLite（结构化） + Ruflo HNSW（语义）
↓
技能执行追踪 → 模式识别 → 置信度衰减 → 程序优化建议
```

**关键创新**:
1. **技能学习**: 记录工具调用，识别高频模式
2. **轨迹追踪**: SONA（Start-Operation-Note-Action）生命周期
3. **程序优化**: 基于执行历史提出改进建议

### 技能学习流程
```
工具调用 → 记录参数+结果+耗时 → 聚类相似调用 → 识别模式 → 提出程序化建议
```

### 与 yaoyao 对比

| 维度 | Memory Unified | yaoyao |
|------|----------------|--------|
| 技能学习 | ✅ 自动识别模式 | ❌ 无 |
| 轨迹追踪 | ✅ SONA | ❌ 仅时间线 |
| 程序建议 | ✅ 主动优化 | ❌ 无 |
| 双后端 | SQLite + HNSW | FTS5 + sqlite-vec |

### yaoyao 可借鉴
- **工具追踪**: 记录 memory_search / memory_save 调用模式
- **高频识别**: "用户经常搜索项目 X" → 主动建议
- **执行优化**: 识别慢查询 → 建议索引优化

## 八、关键洞察

1. **图结构是趋势**: 5/6 个项目使用图，yaoyao 缺失
2. **原子提取是标准**: 不再保存原始对话，提取结构化事实
3. **动态整合优于静态**: 记忆应随新信息演化，而非固定
4. **查询自适应是方向**: 不同查询需要不同检索策略
5. **技能学习是差异化**: 只有 Memory Unified 做，是机会
6. **基准测试是门槛**: 没有 benchmark 难以证明优势
